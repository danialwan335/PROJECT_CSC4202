# 6.1 Correctness of Algorithm

## 6.1.1 Problem Overview

The **Single-Source Shortest Path problem** on a weighted undirected graph G = (V, E, w) with non-negative weights asks: for a given source s, find the shortest path to every other vertex v ∈ V.

We prove correctness of Dijkstra's Algorithm using a **Loop Invariant** on the main relaxation loop.

The algorithm maintains:
- `dist[v]` — current shortest known distance from source s to v
- `prev[v]` — predecessor of v on the current best path
- `visited[]` — boolean set of "settled" vertices whose shortest distance is finalised

## 6.1.2 Loop Invariant

**Statement:** At the beginning of each iteration of the main while-loop, for every vertex u in the visited set S:

> `dist[u]` equals the true shortest path distance δ(s, u) from source s to u.

### Initialisation (Before First Iteration)

Before the loop begins:
- `dist[s] = 0`. Since there are no negative edges, the shortest path from s to itself is indeed 0. ✓
- `visited` set S is empty — the invariant holds vacuously (no elements to violate it). ✓

### Maintenance (During Each Iteration)

Suppose the invariant holds for all vertices in S before an iteration. We extract the vertex u with the minimum distance estimate from PQ.

**Claim:** `dist[u] = δ(s, u)` at the time u is extracted.

**Proof:** Suppose for contradiction that `dist[u] > δ(s, u)`. Then there exists a shorter path P from s to u. Since s ∈ S (source was settled first) and u ∉ S, path P must cross the boundary from S to V\S at some edge (x, y) where x ∈ S and y ∉ S.

By our invariant applied to x: `dist[x] = δ(s, x)`.
When x was settled, edge (x, y) was relaxed, so: `dist[y] ≤ dist[x] + w(x,y) = δ(s,x) + w(x,y)`.

Since all weights are non-negative, the path from y to u adds only non-negative cost:
`δ(s, y) ≤ δ(s, u)`

But u was selected from PQ as the minimum: `dist[u] ≤ dist[y]`.

Therefore: `dist[u] ≤ dist[y] ≤ δ(s, y) ≤ δ(s, u)`.

This contradicts our assumption that `dist[u] > δ(s, u)`. ∴ `dist[u] = δ(s, u)`. ✓

After settling u, all its neighbours v are relaxed:
- If `dist[u] + w(u,v) < dist[v]`: update `dist[v]` and `prev[v]`.
- This maintains the invariant property for future iterations. ✓

### Termination

When the while-loop terminates (PQ empty), all reachable vertices have been settled. For each settled vertex u: `dist[u] = δ(s, u)` by the invariant. For unreachable vertices: `dist[v] = ∞` = δ(s, v). ✓

## 6.1.3 Correctness of Path Reconstruction

The `prev[]` array records, for each settled vertex v, the predecessor u on the shortest path. By induction on the number of hops:
- Base case: `prev[s] = -1` (source has no predecessor).
- Inductive step: When `prev[v]` is set to u, the invariant guarantees `dist[u] = δ(s, u)` and `dist[v] = dist[u] + w(u,v) = δ(s, v)`.

Therefore, backtracking from destination d through `prev[]` yields a path of total weight `δ(s, d)` — the true shortest path. ✓

## 6.1.4 Conclusion

Using the **Loop Invariant technique**, we have proven that:
1. ✓ **Initialisation:** The invariant holds before the first iteration.
2. ✓ **Maintenance:** The invariant is preserved by each relaxation step.
3. ✓ **Termination:** When the loop ends, all distances are optimal.
4. ✓ **Path reconstruction** via `prev[]` correctly identifies the optimal route.

Therefore, **Dijkstra's Algorithm as implemented is correct** and will always produce the minimum-time evacuation route for the landslide scenario, provided all edge weights are non-negative.

> **Note on Our Scenario:** All edge weights (road travel times) are strictly positive, satisfying Dijkstra's non-negativity requirement. The correctness proof applies fully.
