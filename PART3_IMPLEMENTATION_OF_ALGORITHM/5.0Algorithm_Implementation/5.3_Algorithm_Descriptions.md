# 5.3 Algorithm Descriptions

## Overview

This project uses **Dijkstra's Single-Source Shortest Path Algorithm** implemented with a **min-priority queue (binary heap)** to find the fastest ambulance route through a post-landslide road network modelled as a weighted undirected graph.

---

## Key Components

### 1. Graph Representation (Adjacency List)

The road network is stored as an adjacency list: `adj[u]` contains a list of pairs `(v, w)` for every road segment from junction `u` to junction `v` with travel time `w` minutes. This is space-efficient for a sparse network (few surviving roads) and allows O(degree) edge access during relaxation.

### 2. Initialisation

- `dist[source] = 0` — the hospital is 0 minutes from itself.
- `dist[v] = ∞` for all other nodes — initially, no path is known.
- `prev[v] = -1` — no predecessor recorded yet.
- All nodes are inserted into the min-priority queue with their initial distance.

### 3. Greedy Node Extraction

At each step, the node `u` with the **smallest current distance estimate** is extracted from the priority queue. This greedy choice is key: because all weights are non-negative, once `u` is extracted, `dist[u]` is final and cannot be improved by any future path through unvisited nodes.

### 4. Edge Relaxation (Core Operation)

For each neighbour `v` of the extracted node `u`:

> **Relaxation:** if `dist[u] + w(u,v) < dist[v]`, then update `dist[v] = dist[u] + w(u,v)` and `prev[v] = u`.

This propagates the shortest known distance outward from the source, level by level. Each relaxation reduces a node's distance estimate only if a genuinely shorter path has been found.

### 5. Stale Entry Management

When `dist[v]` is updated, a new entry `(dist[v], v)` is added to the priority queue. Old entries with larger distances for the same node become stale. These are detected and skipped using the `visited[]` boolean array — a standard **lazy deletion** technique that avoids the complexity of a full `decreaseKey()` operation.

### 6. Path Reconstruction (Backtracking)

After Dijkstra's completes, the `prev[]` array encodes the shortest path tree. To reconstruct the path from hospital (0) to village (7):
- Start at node 7, follow `prev[7] → prev[...] → 0`.
- Prepend each node to build the path in order from source to destination.

---

## Advantages of This Implementation

- **Correctness:** Proven optimal for non-negative weighted graphs (Dijkstra, 1959).
- **Efficiency:** O((V + E) log V) with a binary heap — sufficient for real-time emergency dispatch.
- **Adaptability:** Removing blocked roads from the graph automatically re-routes around them without changing the algorithm.
- **Edge Cases Handled:** No-path scenario (isolated village) correctly returns ∞ distance with appropriate output.
