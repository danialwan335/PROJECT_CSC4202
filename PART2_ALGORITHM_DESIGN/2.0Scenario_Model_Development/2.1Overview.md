# 2.1 Overview

## Scenario Model Development

The landslide medical evacuation problem is modelled as a **single-source shortest path problem on a weighted undirected graph**. This model captures the essential features of the real-world post-landslide road network in Hulu Selangor.

### Graph Representation

The road network is abstracted into the following components:

| Graph Element | Real-World Counterpart |
|---|---|
| **Vertex (Node)** | A road junction, landmark, or village |
| **Edge** | A surviving road segment connecting two junctions |
| **Edge Weight** | Estimated ambulance travel time (in minutes) on that road segment |
| **Source Node (s)** | Hospital Kuala Kubu Bharu |
| **Destination Node (d)** | Kampung Sungai Lui (patient's village) |

### Graph Properties

- **Undirected:** Roads can be traversed in either direction (ambulance going in, patient being transported out use the same road segments).
- **Weighted:** Each road segment has a non-negative travel time weight reflecting road conditions, gradient, distance, and post-landslide damage.
- **Sparse:** After the landslide, not all roads remain passable — the surviving network is a subset of the original road graph.
- **Connected (assumed for reachability):** At least one path from hospital to village must exist for a solution to be found.

### Two-Component Model

The scenario model breaks down into two distinct computational tasks:

1. **Network Representation:** Encode the surviving road network as a graph with accurately weighted edges.
2. **Optimal Route Computation:** Apply Dijkstra's Algorithm to find the minimum-time path from hospital to village through this network.

The model focuses entirely on **minimising total travel time** — it does not optimise for fuel, road condition comfort, or vehicle type. Time is the sole optimisation criterion in a medical emergency.
