# 3.1 Overview

## Algorithm Design: Dijkstra's Shortest Path

The optimised evacuation route algorithm consists of two main components:

1. **Graph Construction:** Model the post-landslide road network as a weighted adjacency list.
2. **Shortest Path Computation:** Apply Dijkstra's Algorithm with a min-priority queue to find the fastest route from hospital to village.

### Problem Formulation

We model this as a bounded optimisation problem:
- Each **road junction or landmark** is a vertex v ∈ V.
- Each **surviving road segment** is an edge (u, v) ∈ E with non-negative weight w(u, v) = travel time in minutes.
- The **source** is the hospital node s.
- The **destination** is the patient's village node d.
- The **goal** is to find a path P from s to d such that the sum of edge weights along P is minimised.

### Key Modelling Elements

| Element | Description |
|---|---|
| **Vertex** | Road junction, village, or named landmark |
| **Edge** | Surviving road segment |
| **Weight** | Travel time in minutes (non-negative) |
| **Source (s)** | Hospital Kuala Kubu Bharu (Node 0) |
| **Destination (d)** | Kampung Sungai Lui (Node 7) |

### Algorithm Steps

1. **Initialise** all node distances to infinity (∞), except the source which is set to 0.
2. **Insert** all nodes into a min-priority queue keyed by their current distance estimate.
3. **Relax:** Extract the minimum-distance node u from the queue. For each neighbour v of u, if `dist[u] + w(u, v) < dist[v]`, update `dist[v]` and record `prev[v] = u`.
4. **Repeat** until the destination node is extracted or the queue is empty.
5. **Reconstruct** the shortest path by backtracking through the `prev[]` array from destination to source.

### Recurrence / Relaxation Function

The core recurrence relation (relaxation step) is:

> **dist[v] = min(dist[v], dist[u] + w(u, v))**

This is applied for every edge (u, v) when node u is processed. It ensures that whenever a shorter path to v is discovered through u, the distance estimate is updated.

### Optimality Guarantee

Dijkstra's Algorithm is a **greedy algorithm** that exploits the **optimal substructure** of shortest paths: any subpath of a shortest path is itself a shortest path. By always processing the node with the current minimum known distance, the algorithm guarantees that once a node is extracted from the priority queue, its distance estimate is final and optimal.
