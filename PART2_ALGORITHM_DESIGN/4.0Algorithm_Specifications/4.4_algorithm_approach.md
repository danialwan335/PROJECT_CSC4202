# 4.4 Algorithm Approach

## 4.4.1 Algorithm Type

The algorithm uses **Dijkstra's Algorithm** — a **greedy graph algorithm** — to solve the **Single-Source Shortest Path (SSSP)** problem on a weighted undirected graph with non-negative edge weights.

The implementation uses a **min-priority queue (binary heap)** to efficiently select the unvisited node with the smallest current distance estimate at each step.

## 4.4.2 Why Dijkstra's Algorithm?

Dijkstra's Algorithm is chosen because:

1. **Optimality Guarantee:** It guarantees the globally shortest path, not merely a locally good one. This is essential in a life-critical medical emergency where the best route — not a merely acceptable route — is required.

2. **Correctness for Non-Negative Weights:** All edge weights in our scenario (travel times) are non-negative. This satisfies Dijkstra's fundamental requirement. Bellman-Ford would also be correct but is 3–4 times slower (O(VE) vs O((V+E)log V)).

3. **Efficiency:** With a binary min-heap priority queue, Dijkstra's runs in O((V+E) log V) time. For a sparse post-landslide road network with ~8 nodes and ~9 edges, this is essentially instantaneous — critical for real-time emergency dispatch.

4. **Natural Fit for Graph Problems:** The road network is inherently a graph. Dijkstra's Algorithm is purpose-built for exactly this type of problem — unlike sorting or divide-and-conquer paradigms, which would require significant adaptation.

5. **Practical Use:** Dijkstra's Algorithm is the backbone of real-world GPS navigation systems (including Google Maps and Waze), validating its effectiveness for road network shortest path problems.

## 4.4.3 Which Part Requires Recurrence?

The **edge relaxation step** is the recurrence relation in Dijkstra's Algorithm:

> **dist[v] = min(dist[v], dist[u] + w(u, v))**

This is applied repeatedly — for each extracted node u, for each of its neighbours v — building the shortest path solution incrementally from smaller subproblems (shortest paths to closer nodes) toward the complete solution (shortest path to the destination).

## 4.4.4 Function for Optimisation

The **optimisation function** is the minimum total travel time:

> **Optimise:** min Σ w(vᵢ, vᵢ₊₁) over all paths from s to d

Dijkstra's Algorithm achieves this by always expanding the path with the smallest cumulative weight first (greedy selection via min-heap), ensuring the global optimum is reached.
