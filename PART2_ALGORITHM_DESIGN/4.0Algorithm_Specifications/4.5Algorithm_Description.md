# 4.5 Algorithm Description

## 4.5.1 Key Idea

Dijkstra's Algorithm maintains a set of visited nodes S whose shortest distances from the source have been finalised, and a min-priority queue PQ of remaining nodes keyed by their current best distance estimate.

At each step, the node u with the smallest tentative distance is extracted from PQ and "settled" (its distance is finalised). All edges from u to unvisited neighbours v are then relaxed. This greedy approach works because:
- All edge weights are non-negative, so the distance to any node can only increase as we look further.
- The node with the smallest tentative distance cannot be improved by any future path through unvisited nodes (which all have larger or equal distances).

## 4.5.2 Recurrence Relation

For each node u extracted from PQ, and for each edge (u, v, w):

```
if dist[u] + w(u, v) < dist[v]:
    dist[v] ← dist[u] + w(u, v)
    prev[v] ← u
    update priority of v in PQ
```

## 4.5.3 Algorithm Steps

### Step 1: Preprocessing — Build the Graph
- Read all road junctions (nodes) and surviving road segments (edges with weights).
- Build an adjacency list representation of the graph.

### Step 2: Initialisation
- Set `dist[s] = 0` for the source node (hospital).
- Set `dist[v] = ∞` for all other nodes v.
- Set `prev[v] = -1` (undefined) for all nodes.
- Insert all nodes into a min-priority queue PQ with their initial distances.

### Step 3: Main Loop — Relaxation
```
while PQ is not empty:
    u ← PQ.extractMin()      // Extract node with smallest dist
    if u == destination: break  // Optimal path found
    for each (v, w) in adjacencyList[u]:
        if dist[u] + w < dist[v]:
            dist[v] = dist[u] + w
            prev[v] = u
            PQ.decreaseKey(v, dist[v])  // Update priority
```

### Step 4: Path Reconstruction — Backtracking
```
path ← empty list
current ← destination
while current ≠ -1:
    path.prepend(current)
    current ← prev[current]
return path, dist[destination]
```

## 4.5.4 Key Data Structures

| Data Structure | Purpose |
|---|---|
| Adjacency List | Store graph edges and weights efficiently |
| dist[] array | Track current shortest distance to each node |
| prev[] array | Track predecessor on shortest path (for reconstruction) |
| Min-Priority Queue | Efficiently select the next node with minimum distance |

## 4.5.5 Termination Condition

The algorithm terminates when:
1. The destination node is extracted from PQ (optimal solution found), or
2. PQ becomes empty (destination is unreachable — no path exists).
