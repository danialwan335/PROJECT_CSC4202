# 4.2 Input and Output Specification

## 4.2.1 Input

- `V`: Number of vertices (road junctions/landmarks). In our scenario, V = 8 (nodes 0–7).
- `E`: List of edges, each represented as a triple `(u, v, w)` where:
  - `u` = source node ID of road segment
  - `v` = destination node ID of road segment
  - `w` = travel time in minutes (non-negative integer)
- `s`: Source node ID = 0 (Hospital Kuala Kubu Bharu)
- `d`: Destination node ID = 7 (Kampung Sungai Lui)

### Sample Input (Edge List)

```
8 9
0 1 8
1 2 10
1 3 18
2 4 20
2 5 36
3 5 17
4 6 10
5 7 12
6 7 14
Source: 0
Destination: 7
```

## 4.2.2 Output

- **Shortest Travel Time:** The minimum total time (in minutes) to travel from node s to node d.
- **Optimal Route:** The ordered list of node IDs from s to d forming the shortest path.
- **Segment-by-Segment Breakdown:** For each edge in the path, the segment label and travel time.

### Sample Output

```
=== LANDSLIDE EVACUATION ROUTE OPTIMISER ===
Source     : 0 (Hospital Kuala Kubu Bharu)
Destination: 7 (Kampung Sungai Lui)

Shortest Travel Time: 55 minutes

Optimal Route: 0 → 1 → 3 → 5 → 7

Segment Breakdown:
  Node 0 → Node 1 :  8 min  [Hospital KKB → Pekan KKB Junction]
  Node 1 → Node 3 : 18 min  [Pekan KKB → Sg. Chilling Entrance]
  Node 3 → Node 5 : 17 min  [Sg. Chilling → Pos Gertak]
  Node 5 → Node 7 : 12 min  [Pos Gertak → Kg. Sungai Lui]
  -----------------------------------------------
  Total            : 55 min
```
