# 4.3 Mathematical Model

## 4.3.1 Graph Definition

Let G = (V, E, w) be a weighted undirected graph where:
- V = {0, 1, 2, 3, 4, 5, 6, 7} (set of vertices representing road junctions)
- E ⊆ V × V (set of surviving road segment edges)
- w : E → ℝ⁺₀ such that w(u, v) ≥ 0 ∀ (u, v) ∈ E (non-negative travel time weights)

## 4.3.2 Shortest Path Optimisation

**Objective Function:**

Find path P* = (s = v₀, v₁, v₂, ..., vₖ = d) through G such that:

> **minimise** d(s, d) = Σᵢ₌₀ᵏ⁻¹ w(vᵢ, vᵢ₊₁)

subject to:
- (vᵢ, vᵢ₊₁) ∈ E for all i = 0, 1, ..., k-1  *(each step uses a real surviving road)*
- No node is visited more than once  *(simple path — no cycles)*
- w(u, v) ≥ 0 for all (u, v) ∈ E  *(non-negative weights — required for Dijkstra)*

## 4.3.3 Relaxation Equation

The core mathematical operation is the **edge relaxation** formula:

> **dist[v] = min(dist[v], dist[u] + w(u, v))**

This is applied for each edge (u, v) when node u is processed. It encodes Bellman's **principle of optimality**: the optimal path to any node v consists of an optimal path to some predecessor u plus the direct edge from u to v.

## 4.3.4 Optimal Substructure Property

**Theorem:** If P* = (s, ..., u, v, ..., d) is a shortest path from s to d, then the subpath (s, ..., u) is a shortest path from s to u.

**Proof sketch:** If there existed a shorter path from s to u, say Q, then replacing (s,...,u) in P* with Q would yield a shorter total path from s to d, contradicting the optimality of P*. ∎

This property guarantees that Dijkstra's Algorithm — which builds optimal paths incrementally — is correct.

## 4.3.5 Distance Labels

Dijkstra's Algorithm maintains a distance label dist[v] for each vertex v ∈ V:
- Initially: dist[s] = 0, dist[v] = ∞ for all v ≠ s
- Throughout: dist[v] represents the **length of the shortest known path** from s to v
- Upon termination: dist[v] holds the **true shortest path distance** from s to v

## 4.3.6 Predecessor Array

A predecessor array `prev[]` records the optimal path:
- prev[v] = u means "on the optimal path to v, the node before v is u"
- The optimal path is reconstructed by tracing prev[] from d back to s
